01 / THE TURN

I built the counter first.
It exposed the better product.

The hackathon version counted prompts, responses, copies, and corrections. That made activity visible, but activity was not the thing worth carrying between providers.

The durable value was behavioral: how someone researches, what they verify, what they delegate, and what “done” means to them. Provider history can remember that, but the user cannot take it elsewhere. DataSelf moved from an interaction ledger to a portable memory layer.

Product insight The useful unit is not a prompt. It is a reusable working rule.
02 / USER + JOB

One user.
One painful reset.

First user

An individual whose work with an LLM improves as the model learns how they operate.

Job to be done

When I change LLMs, help me bring how I work so I do not have to teach the new one from zero.

Why not teams first?

Teams add permissions, shared truth, and governance before the core transfer is proven. Personal memory gives the shortest path to a falsifiable test.

03 / PRODUCT DECISIONS

Choosing what not to build
was most of the product work.

Each choice bought speed or trust and accepted a visible cost.

Memory format

Behavioral rules over transcripts or embeddings

Portable and legible without retaining the most sensitive material.

Cost: loses conversational nuance.
Extraction

Active LLM over sending the chat to a second model

The provider already has the session, so extraction needs no second copy.

Cost: adapter quality depends on provider UI.
Storage

Local Markdown over a cloud account

The user can inspect, edit, move, or delete every memory with ordinary tools.

Cost: no automatic multi-device sync.
Update loop

Automatic + provisional over approve-every-item

A menu-bar tool cannot become another inbox. Repetition strengthens a memory; edits remain reversible.

Cost: weak inferences can appear temporarily.
Portability

One context file over provider integrations

Markdown works everywhere and made provider transfer testable in one week.

Cost: the first migration step is manual.
Explicit non-goals

No cloud sync. No team memory. No raw chat archive.

Those features widen the system before proving that portable memory changes the next session.

04 / MVP

A small loop that ends
in a portable artifact.

The output stays useful even if DataSelf disappears.

  1. 01Start

    Toggle recording from the macOS menu bar.

  2. 02Work

    Use the active LLM normally; retain only content-free events.

  3. 03Extract

    Ask the active LLM for a strict memory patch at session end.

  4. 04Validate

    Reject unknown types, multiline content, or mismatched evidence.

  5. 05Move

    Export dataself-context.md and bring it to a new LLM.

05 / TEST

I tested behavior,
not whether the file opened.

Two fresh anonymous ChatGPT sessions received the same vague new product request. One had no memory. One received DataSelf’s context.

Without DataSelf

Selected a product direction, proposed a broad multi-week build, and asked several questions at once.

With DataSelf

Made no product assumption, asked one focused question, kept the MVP narrow, and reserved execution until direction was clear.

Result 4 of 5 known behaviors transferred without correction.

The fifth—three-line completion handoff—was not exercised because the test stopped at product definition.

07 / WHAT I KNOW

Built evidence is not
market evidence.

The prototype proves the memory can transfer. It does not yet prove that enough people feel the reset, trust automatic extraction, or keep using the system.

Next experiment

Recruit five heavy LLM users for three sessions each. Measure setup time, memory edit/deletion rate, corrective messages with and without context, and whether they choose to migrate the file again.

Risks I would watch

  • Provider UI changes breaking automatic capture.
  • Confident but wrong memories becoming sticky.
  • A context file growing until it becomes noise.
  • Users valuing convenience more than portability.

Product definition · prioritization · privacy · prototyping · experiment design · metrics · technical execution

DataSelf is small on purpose.
The question it tests is not.
